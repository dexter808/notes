# Databases

A database is an organized collection of data designed for efficient management. It allows for CRUD operations.

## Categories of DBs

### SQL / Relational DBs

- Data is stored in fixed schema / design of tables rows and columns. 
- Evolved over 40 years of operation
- Supports ACID Transactions.
- Complex Operations like joins are supported. 

### NoSQL / Non-Relational DBs

- Data can be stored in some flexible structure or design. Like Documents, Json, Graph etc.
- Newer but provides better latency and throughput.
- Support Eventual Consistency / BASE (Basically Availability, Soft state and Eventual Consistency)
- Data recieved might not be accurate always but eventually it will become accurate.

## Advantages of DBs

- Large Data Management
- Data consistency
- Efficient Updates - Faster DML operation
- Security - Authorization for users
- Data Integrity - Constraints can be enforced to ensure data integrity across transactions
- Availability - Available to multiple users and machines across servers
- Scalibility - Can grow or scale as required.
